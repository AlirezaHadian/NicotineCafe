# نیکوتین کافه — Nicotine Cafe

سیستم تشخیص خودکار محصول از روی صدا (فارسی، آفلاین، بدون دکمه) + نمایشگر محصول WPF
برای فروشگاه دخانیات. مشتری اسم محصول رو می‌گه، سیستم خودش گوش می‌ده، تشخیص می‌ده،
و عکس/مشخصات محصول رو نشون می‌ده.

## ساختار سولوشن

```
NicotineCafe/
├── NicotineCafe.sln
├── database/
│   ├── schema.sql              # ساختار جداول SQLite (منبع مشترک دو پروژه)
│   ├── seed.sql                 # داده‌ی نمونه (۱۶ برند / ۴۸ محصول)
│   └── nicotinecafe.db.sample   # دیتابیس ساخته‌شده‌ی نمونه (کپی کن به database\nicotinecafe.db)
├── src/
│   ├── NicotineCafe.Core/       # مدل‌ها + اینترفیس‌ها (بدون وابستگی به دیتابیس/UI)
│   ├── NicotineCafe.Data/       # پیاده‌سازی Repository روی SQLite (Dapper)
│   ├── NicotineCafe.Services/   # لایه سرویس + کلاینت TCP موتور صدا
│   └── NicotineCafe.WPF/        # اپلیکیشن WPF (MVVM, DI)
│       ├── MainWindow           # صفحه‌ی اصلی نمایش محصول (بدون دکمه — گوش‌به‌زنگ خودکار)
│       └── Views/AdminWindow    # مدیریت محصولات + تنظیمات موتور صدا
└── voice-engine/                 # موتور تشخیص گفتار پایتون (نسخه Production)
```

## معماری لایه‌ها (WPF)

```
IProductRepository       ──►  SqliteProductRepository     (فقط کوئری خام اینجاست)
IProductService           ──►  ProductService               (منطق کسب‌وکار + کش)
IEngineSettingsRepository ──►  SqliteEngineSettingsRepository (تنظیمات موتور صدا، کلید-مقدار)
ISpeechEngineClient       ──►  SpeechEngineClient            (اتصال TCP به موتور پایتون)
```

هر بخشی که کوئری SQL داره فقط داخل `NicotineCafe.Data` هست و از اینترفیس‌های
بالا ارث‌بری می‌کنه. UI هیچ‌وقت مستقیم SQL نمی‌زنه.

## نحوه‌ی کار (بدون دکمه!)

موتور پایتون به‌صورت دائمی به میکروفون گوش می‌ده (`AutoVadRecorder` — تشخیص
خودکار شروع/پایان گفتار بر اساس انرژی صدا). مشتری فقط حرف می‌زنه؛ سیستم خودش
تشخیص می‌ده گفتار کِی شروع و کِی تموم شده، صدا رو ضبط می‌کنه، متن رو استخراج
می‌کنه (Whisper)، محصول رو پیدا می‌کنه (fuzzy/phonetic matching روی برند و رنگ)،
و نتیجه رو از طریق یک سوکت TCP محلی به WPF می‌فرسته.

```
WPF  ⇄  127.0.0.1:8765  ⇄  voice-engine (Python)
```

پیام‌های موتور به WPF (یک‌طرفه، JSON خط‌به‌خط):
```json
{"type":"audio_level","level":0.42}
{"type":"status","message":"processing"}
{"type":"recognition","isValid":true,"productId":1,"rawText":"...","confidence":0.92}
```

پیام‌های WPF به موتور (فقط برای توقف/ازسرگیری):
```json
{"type":"pause"}
{"type":"resume"}
```

## قانون نمایش محصول (۵ دقیقه)

در `MainViewModel`:
- با تشخیص هر محصول معتبر، تایمر ۵ دقیقه‌ای ری‌ست می‌شه.
- محصول قبل از ۵ دقیقه فقط در دو حالت پنهان می‌شه:
  1. یک محصول جدید تشخیص داده بشه (جایگزین می‌شه و تایمر دوباره شروع می‌شه)
  2. اپراتور روی دکمه‌ی ✕ کلیک کنه

## رابط کاربری

- **بدون دکمه‌ی میکروفون** — یک «ارب شنیداری» (حلقه‌های هم‌مرکز طلایی) وسط صفحه
  به‌آرومی نفس می‌کشه و با صدای واقعی واکنش نشون می‌ده؛ صرفاً نمایشیه.
- نوار بالا (LTR، مثل ویندوز معمولی): لوگو/نام سمت چپ؛ وضعیت اتصال، دکمه‌ی
  توقف موقت/ازسرگیری شنیدن، دکمه‌ی ⚙ (مدیریت محصولات)، و دکمه‌های کوچک‌کردن/
  بستن سمت راست (چون پنجره `WindowStyle="None"` داره و کروم پیش‌فرض ویندوز رو نداره).
- پنجره‌ی مدیریت (⚙): دو تب — «محصولات» (افزودن/ویرایش/حذف) و «تنظیمات موتور
  تشخیص گفتار» (مدل، Beam Size، آستانه‌ها و...).

## فونت‌ها

پوشه‌ی `src/NicotineCafe.WPF/Assets/Fonts/` رو با فونت **Vazirmatn** (فارسی+لاتین،
لایسنس آزاد، طراحی مدرن هندسی) پر کن. فایل‌های `.ttf` رو همون‌جا بذار؛
`Themes/Fonts.xaml` از قبل بهشون رفرنس می‌ده.

## عکس محصولات

مسیر عکس هر محصول توی ستون `ImagePath` جدول `Products` ذخیره می‌شه (نسبی، مثل
`Images/winston_blue.png`). خود فایل‌های عکس رو باید توی
`src/NicotineCafe.WPF/Assets/Images/` بذاری — اسم فایل باید دقیقاً با چیزی که
توی دیتابیس (یا پنجره‌ی مدیریت، فیلد «مسیر عکس») نوشتی یکی باشه.

## راه‌اندازی

### ۱) دیتابیس
```powershell
copy database\nicotinecafe.db.sample database\nicotinecafe.db
```
یا با `schema.sql` + `seed.sql` خودت از صفر بساز.

### ۲) موتور صدا
```powershell
cd voice-engine
pip install -r requirements.txt
python main.py serve --port 8765 --db ..\database\nicotinecafe.db
```
(دستی اجراکردن فقط برای دیباگ لازمه — WPF خودش موتور رو استارت می‌کنه.)

### ۳) WPF
سولوشن رو با Visual Studio 2022 باز کن، `NicotineCafe.WPF` رو Startup Project کن، Run.

## عیب‌یابی

- **موتور صدا بالا نمیاد**: فایل `voice-engine.log` کنار `NicotineCafe.exe` رو
  چک کن — هر خطایی اونجا با timestamp ثبت می‌شه.
- **پورت اشغاله (bind failed)**: یعنی یه `python.exe` قدیمی از اجرای قبلی هنوز
  زنده‌ست. توی Task Manager همه‌ی `python.exe` رو ببند و دوباره Run کن.
- **دقت تشخیص پایینه**: اول فیلدهای دیباگ (`debugBrand`, `debugBrandScore`,
  `debugVariant`, `debugVariantScore`) توی `voice-engine.log` رو ببین — نشون
  می‌ده دقیقاً کجا (برند یا رنگ) گیر کرده. بعد این‌ها رو امتحان کن:
  1. توی پنجره‌ی مدیریت → تب تنظیمات، مدل رو از `tiny` به `small` عوض کن
     (دقت بیشتر، سرعت کمتر).
  2. الیاس‌های بیشتری به `BrandAliases`/`VariantAliases` (توی `database/seed.sql`
     یا مستقیم توی دیتابیس) اضافه کن — هرچی که واقعاً از مشتری‌ها می‌شنوی.
- **سرعت پایینه**: تب تنظیمات → `Beam Size` رو روی ۱ نگه دار، مدل `tiny` رو
  امتحان کن، و `تعداد هسته‌ی CPU` رو با سخت‌افزار خودت تنظیم کن (عدد خیلی
  بزرگ گاهی کندتر می‌کنه، نه سریع‌تر).
