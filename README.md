# نیکوتین کافه — Nicotine Cafe

سیستم تشخیص خودکار **برند** از روی صدا (فارسی، آفلاین، بدون دکمه) + نمایشگر
WPF برای فروشگاه دخانیات. مشتری اسم برند رو می‌گه (با هر مدل/رنگی)، سیستم
خودش گوش می‌ده، برند رو تشخیص می‌ده، و عکس برند + لیست مدل‌های موجود رو نشون
می‌ده.

## معماری (v2 — برند-محور)

تشخیص گفتار فقط **برند** رو مشخص می‌کنه، نه مدل/رنگ خاص. اگه مشتری بگه
«وینستون آبی» یا «وینستون لایت» یا فقط «وینستون»، در هر سه حالت همون کارت
برند «وینستون» نمایش داده می‌شه (عکس برند + اسم)، و اگه فعال باشه، زیرش
لیست مدل‌هایی که مغازه از اون برند داره («وینستون آبی»، «وینستون قرمز»، ...)
به‌صورت برچسب‌های شیک نشون داده می‌شه — هیچ مشخصات فنی (قطران/نیکوتین) دیگه
نمایش داده نمی‌شه.

## ساختار سولوشن

```
NicotineCafe/
├── NicotineCafe.sln
├── database/
│   ├── schema.sql              # Brands / BrandAliases / BrandModels / EngineSettings
│   ├── seed.sql                 # ۲۴ برند نمونه (اسم‌ها واقعی، عکس/مدل‌ها placeholder)
│   └── nicotinecafe.db.sample   # دیتابیس ساخته‌شده‌ی نمونه
├── src/
│   ├── NicotineCafe.Core/       # مدل‌ها + اینترفیس‌ها
│   ├── NicotineCafe.Data/       # SqliteBrandRepository, SqliteEngineSettingsRepository
│   ├── NicotineCafe.Services/   # BrandService، کلاینت TCP موتور صدا
│   └── NicotineCafe.WPF/        # اپلیکیشن WPF (MVVM, DI)
│       ├── MainWindow           # صفحه‌ی اصلی نمایش برند (بدون دکمه)
│       └── Views/AdminWindow    # مدیریت برند/الیاس/مدل + تنظیمات موتور صدا
└── voice-engine/                 # موتور تشخیص برند، پایتون (Production)
```

## معماری لایه‌ها (WPF)

```
IBrandRepository          ──►  SqliteBrandRepository          (فقط کوئری خام اینجاست)
IBrandService              ──►  BrandService                    (کش + منطق کسب‌وکار)
IEngineSettingsRepository ──►  SqliteEngineSettingsRepository (تنظیمات، کلید-مقدار)
ISpeechEngineClient       ──►  SpeechEngineClient              (اتصال TCP به موتور پایتون)
```

## نحوه‌ی کار (بدون دکمه!)

موتور پایتون دائم گوش می‌ده (`AutoVadRecorder`)؛ مشتری فقط حرف می‌زنه. متن با
Whisper استخراج می‌شه، **فقط برند** با fuzzy/phonetic matching پیدا می‌شه
(دیگه تشخیص رنگ/مدل لازم نیست)، و `BrandId` روی یک سوکت TCP محلی به WPF
فرستاده می‌شه.

```
WPF  ⇄  127.0.0.1:8765  ⇄  voice-engine (Python)
```

```json
{"type":"recognition","isValid":true,"brandId":2,"rawText":"...", "confidence":0.92}
{"type":"audio_level","level":0.42}
{"type":"status","message":"processing"}
```

WPF فقط برای توقف/ازسرگیریِ شنیدن به موتور پیام می‌فرسته:
```json
{"type":"pause"}
{"type":"resume"}
```

## قانون نمایش (۵ دقیقه)

با هر تشخیص معتبر، تایمر ۵ دقیقه‌ای ری‌ست می‌شه. قبل از ۵ دقیقه فقط در دو
حالت پنهان می‌شه: (۱) برند جدیدی تشخیص داده بشه، (۲) اپراتور روی ✕ بزنه.

## پنجره‌ی مدیریت (⚙)

سه تب:

1. **برندها** — لیست همه‌ی برندها؛ فرم افزودن/ویرایش (نام فارسی/انگلیسی +
   دکمه‌ی «انتخاب فایل...» برای عکس — فایل انتخابی خودکار کپی می‌شه توی
   `Assets/Images` و مسیرش ذخیره می‌شه، دیگه نیازی به تایپ‌کردن مسیر نیست)؛
   زیرش مدیریت **الیاس‌ها** (تلفظ‌های عامیانه‌ای که موتور صدا باید بشناسه) و
   **مدل‌ها** (چیزهایی که زیر اسم برند روی کارت نمایش نشون داده می‌شه —
   این‌ها کاملاً نمایشی‌ان و روی تشخیص گفتار اثری ندارن).
2. **تنظیمات موتور تشخیص گفتار** — مدل Whisper، Beam Size، آستانه‌های VAD،
   تعداد هسته‌ی CPU، حداقل اطمینان، و یک **چک‌باکس** برای روشن/خاموش‌کردن
   نمایش لیست مدل‌ها زیر اسم برند (`show_model_list`).

همه‌ی این‌ها توی جدول‌های `Brands` / `BrandAliases` / `BrandModels` /
`EngineSettings` همون دیتابیس مشترک ذخیره می‌شن.

## دانلود مدل Whisper — سوال متداول

بله، `faster-whisper` مدل‌ها رو **خودش خودکار دانلود می‌کنه** از Hugging Face
در اولین اجرا با اون اندازه‌ی مدل (`tiny`/`base`/`small` هرکدوم جدا دانلود
می‌شن)، و توی کش لوکال ذخیره می‌کنه:
```
%USERPROFILE%\.cache\huggingface\hub
```
یعنی فقط **بار اول** برای هر مدل نیاز به اینترنت هست؛ بعدش کاملاً آفلاین کار
می‌کنه. اگه سیستم مقصد اصلاً اینترنت نداره، باید از یه سیستم دیگه مدل رو
دانلود کنی و پوشه‌ی کش رو کپی کنی، یا مسیر مدل از‌قبل‌دانلودشده رو مستقیم توی
`App.xaml.cs` → `VoiceEngineProcessLauncher` → `modelPath` بدی.

## راه‌اندازی

### ۱) دیتابیس
```powershell
copy database\nicotinecafe.db.sample database\nicotinecafe.db
```

### ۲) موتور صدا
```powershell
cd voice-engine
pip install -r requirements.txt
python main.py serve --port 8765 --db ..\database\nicotinecafe.db
```
(دستی اجراکردن فقط برای دیباگ لازمه — WPF خودش موتور رو استارت می‌کنه.)

### ۳) WPF
سولوشن رو با Visual Studio 2022 باز کن، `NicotineCafe.WPF` رو Startup Project کن، Run.

## عیب‌یابی

- **موتور صدا بالا نمیاد**: فایل `voice-engine.log` کنار `NicotineCafe.exe`.
- **پورت اشغاله**: یه `python.exe` قدیمی زنده‌ست — توی Task Manager ببندش.
- **دقت پایینه**: تب تنظیمات → مدل رو از `tiny` به `small` عوض کن. الیاس‌های
  بیشتری هم از تب برندها اضافه کن (هرچی که واقعاً از مشتری‌ها می‌شنوی).
- **سرعت پایینه**: `Beam Size=1`, مدل `tiny`, و `تعداد هسته‌ی CPU` رو با
  سخت‌افزار خودت تنظیم کن (عدد خیلی بزرگ گاهی کندتر می‌کنه، نه سریع‌تر).

## بسته‌بندی نهایی — نصب کاملاً آفلاین (بدون نیاز به Python یا اینترنت روی سیستم مغازه)

چهار فایل برای این کار اضافه شده:

```
NicotineCafe/
├── scripts/publish.ps1                    # مرحله ۲: publish خودکفا از WPF
└── installer/
    ├── prepare-python-bundle.ps1          # مرحله ۱: پایتون + پکیج‌ها + مدل رو دانلود می‌کنه
    ├── NicotineCafeSetup.iss              # اسکریپت Inno Setup
    └── build-installer.ps1                # مرحله ۳: کامپایل installer از روی .iss
```

نتیجه‌ی نهایی یه فایل `NicotineCafeSetup.exe` هست که روی سیستم مغازه:
- نیاز به Python نصب‌شده **ندارد** (یک نسخه‌ی خصوصیِ پرتابل داخل خودش داره)
- نیاز به اینترنت **ندارد** (نه برای پکیج‌ها، نه برای مدل Whisper)
- فقط دابل‌کلیک و Next-Next می‌خواد

### مرحله ۱ — آماده‌سازی باندل پایتون (فقط یک‌بار، روی سیستمی با اینترنت)

```powershell
powershell -ExecutionPolicy Bypass -File installer\prepare-python-bundle.ps1
```

این اسکریپت خودش به‌ترتیب:
1. پایتون ۳.۱۱ نسخه‌ی embeddable (پرتابل، بدون نصب) رو دانلود می‌کنه توی `python-embed\`
2. site-packages رو توش فعال می‌کنه
3. pip رو نصب می‌کنه
4. تمام پکیج‌های `voice-engine\requirements.txt` رو **داخل همون پوشه‌ی پرتابل** نصب می‌کنه (نه روی سیستم)
5. مدل Whisper (پیش‌فرض `tiny`) رو دانلود و توی `model-cache\` کش می‌کنه

اگه می‌خوای مدل دیگه‌ای (مثلاً `small`) رو هم آماده کنی:
```powershell
powershell -ExecutionPolicy Bypass -File installer\prepare-python-bundle.ps1 -ModelSize small
```
(اگه مدل رو توی تنظیمات عوض کردی، حتماً همون مدل رو اینجا هم بزن تا از‌قبل کش شده باشه.)

بعد از این مرحله، دو تا پوشه‌ی جدید کنار `NicotineCafe.sln` می‌بینی: `python-embed\` و
`model-cache\` — این‌ها بزرگن (چندصد مگابایت تا ~۱ گیگ بسته به مدل)، طبیعیه.

### مرحله ۲ — Publish

```powershell
powershell -ExecutionPolicy Bypass -File scripts\publish.ps1
```
این خودش `python-embed\` و `model-cache\` رو هم (چون توی csproj تعریف شدن) به
خروجی `publish\NicotineCafe\` کپی می‌کنه.

### مرحله ۳ — ساخت Installer

1. [Inno Setup](https://jrsoftware.org/isdl.php) رو نصب کن (رایگان، اگه قبلاً نصب نکردی).
2. اجرا کن:
   ```powershell
   powershell -ExecutionPolicy Bypass -File installer\build-installer.ps1
   ```
3. فایل نهایی: `installer\Output\NicotineCafeSetup.exe`

همینو به هر سیستمی (حتی بدون اینترنت، بدون Python) منتقل کن و اجرا کن — کار می‌کنه.

### نکات مهم

- مراحل ۱ رو فقط **یک‌بار** لازمه انجام بدی (یا هروقت خواستی مدل رو عوض کنی).
  مرحله‌ی ۲ و ۳ رو هر بار که کد/عکس‌ها عوض شد دوباره بزن.
- قبل از publish نهایی، عکس‌های واقعی برندها رو از پنجره‌ی مدیریت اضافه کن.
- اگه بعداً خواستی مدل رو از `tiny` به `small` عوض کنی: هم توی پنجره‌ی مدیریت
  (تب تنظیمات) عوضش کن، هم `prepare-python-bundle.ps1 -ModelSize small` رو
  دوباره بزن تا مدل جدید هم کش بشه، وگرنه موقع اجرا سعی می‌کنه از اینترنت
  دانلودش کنه و روی سیستم بدون اینترنت fail می‌شه.

## بهبود مداوم دقت (RecognitionLog)

هر تلاش برای تشخیص (چه موفق چه ناموفق) توی جدول `RecognitionLog` ثبت می‌شه:
متن خام، متن نرمالایز‌شده، برند تشخیص‌داده‌شده (اگه بوده)، اطمینان، و اینکه
معتبر بوده یا نه. هر از گاهی این جدول رو با یه SQLite browser (مثل DB
Browser for SQLite) چک کن:

```sql
SELECT RawText, MatchedBrandId, Confidence, IsValid, CreatedAt
FROM RecognitionLog
WHERE IsValid = 0
ORDER BY CreatedAt DESC
LIMIT 50;
```

ردیف‌های `IsValid = 0` یا اطمینان پایین معمولاً نشون می‌دن موتور صدا چه
عبارت‌هایی رو اشتباه شنیده — همون‌ها رو به‌عنوان الیاس جدید (از پنجره‌ی
مدیریت) اضافه کن تا دقت مدام بهتر بشه.

## الیاس‌های ترکیبی (برند + رنگ/تنوع)

چون مشتری‌ها معمولاً برند رو همراه با رنگ/تنوع می‌گن («وینستون آبی»،
«وینستون اولترالایت»)، برای هر ۲۴ برند اولیه، الیاس‌های ترکیبی با ۸ تنوع
رایج (آبی، قرمز، طلایی، نقره‌ای، منتول، اولترالایت، مشکی، سفید) از قبل
اضافه شده — در کنار الیاس‌های ساده‌ی خودِ برند. اگه تنوع دیگه‌ای هم شنیدی که
جزو این لیست نبود، از پنجره‌ی مدیریت اضافه‌ش کن.

## بهینه‌سازی‌های سرعت اعمال‌شده

- مدل پیش‌فرض `tiny` + `beam_size=1` (سریع‌ترین حالت با افت دقت جزئی)
- تشخیص hallucination حین تولید متن (نه بعدش) — جلوی هدررفت چند ثانیه‌ای رو می‌گیره
- `without_timestamps=True` — چون هیچ‌جا از timestamp استفاده نمی‌کنیم، محاسبه‌ش حذف شد
- **Warm-up خودکار مدل** بعد از لود‌شدن (یک استنتاج آزمایشی روی سکوت) — اولین
  درخواست واقعی مشتری دیگه هزینه‌ی اول‌بار (JIT/تخصیص حافظه) رو نمی‌پردازه
- بافرینگ میکروفون (`latency="high"`) برای جلوگیری از افت صدا هنگام مشغول‌بودن CPU
- تعداد هسته‌ی CPU قابل‌تنظیم از پنجره‌ی مدیریت (بیشتر همیشه سریع‌تر نیست)
